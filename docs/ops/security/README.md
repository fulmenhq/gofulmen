# Security Operations

This directory contains operational security documentation for the gofulmen repository, including security assessments, memos, and decision records related to security practices.

## Purpose

- Document security assessments and audits for releases
- Maintain records of security decisions and suppressions
- Provide transparency into security practices for contributors and users

## Contents

- `memos/`: Security assessment memos and notes for specific releases or incidents

## Reporting Security Issues

If you suspect a security issue in gofulmen or related FulmenHQ projects, please follow the reporting guidelines in our [organization security policy](https://github.com/3leaps/SECURITY.md#how-to-report).

This includes:

- Responsible disclosure practices
- Contact information for security reports
- Guidelines for handling vulnerabilities

## Standards

Security documentation follows the [repository safety framework](../crucible-go/standards/repository-safety-framework.md) and [security standards](../crucible-go/standards/security/README.md).
