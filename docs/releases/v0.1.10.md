# Release Notes: v0.1.10

**Release Date**: 2025-11-09  
**Release Type**: Bug Fix & Refactoring  
**Status**: ✅ Released

## Overview

This release migrates signals package from `pkg/signals/` to `signals/` for consistency with top-level module structure, and fixes documentation issues that were breaking downstream template development. This is a focused release to support microtool template users who were experiencing import path failures.

## Breaking Changes

### Signals Import Path Migration

- **Old**: `github.com/fulmenhq/gofulmen/pkg/signals`
- **New**: `github.com/fulmenhq/gofulmen/signals`

### Migration Required

```bash
# Update all Go files importing signals
find . -name "*.go" -exec sed -i 's|github.com/fulmenhq/gofulmen/pkg/signals|github.com/fulmenhq/gofulmen/signals|g' {} \;

# Update documentation
find . -name "*.md" -exec sed -i 's|github.com/fulmenhq/gofulmen/pkg/signals|github.com/fulmenhq/gofulmen/signals|g' {} \;
```

## Migration Guide

### Quick Migration for Existing Projects

1. **Update imports**:

   ```go
   // Replace this
   import "github.com/fulmenhq/gofulmen/pkg/signals"

   // With this
   import "github.com/fulmenhq/gofulmen/signals"
   ```

2. **Update go.mod** (if using replace directives):

   ```go
   // Update any replace directives
   replace github.com/fulmenhq/gofulmen/pkg/signals => github.com/fulmenhq/gofulmen/signals v0.1.10
   ```

3. **Test and verify**:
   ```bash
   go mod tidy
   go test ./...
   go build ./...
   ```

### Template Users

If you were using the microtool template that referenced `pkg/signals`, update your template files or regenerate with the corrected template that will be available after this release.
