## Overview

This release resolves critical Windows build failures in the signals package and updates to Crucible v0.2.11 with comprehensive verification. The Windows fix enables cross-platform compatibility while maintaining Unix functionality, and the Crucible update brings the latest fulpack type generation framework for future cross-language support.

## Key Changes

### ðŸ”§ Windows Build Compatibility Fix

**Problem**: Windows builds were failing because `signals/http.go` referenced `syscall.SIGUSR1/SIGUSR2` which aren't defined on Windows.

**Solution**: Implemented platform-specific signal handling using Go build tags:

- **`signals/platform_signals_unix.go`**: Defines SIGUSR1/2 for Unix systems (Linux, macOS)
- **`signals/platform_signals_windows.go`**: Empty map for Windows compatibility
- **`signals/http.go`**: Updated to use dynamic signal map composition
- **Result**: Maintains full Unix functionality while enabling Windows builds

**Enhanced Verification Process**: Updated with comprehensive verification to prevent dependency drift:

- **`go.mod`**: Updated from v0.2.9 to v0.2.11
- **Verification**: Confirmed via `go list -m github.com/fulmenhq/crucible` shows v0.2.11
- **Sync Config**: Updated `.goneat/ssot-consumer.yaml` to use v0.2.11 ref
- **Hashes**: Confirmed `go.sum` contains v0.2.11 hashes
- **Cleanup**: Removed inconsistent vendor directory

**New Features from Crucible v0.2.11**:

- **Code Generation**: Templates for Python, TypeScript, and Go type generation
- **Future Support**: Foundation for fulpack implementation in gofulmen
