## Overview

Version 0.1.16 updates gofulmen to support Crucible v0.2.17, bringing enhanced DevSecOps secrets schema with structured credential objects, lifecycle metadata, and rotation policies. This enables fulmen-secrets v0.1.1+ enterprise credential management features.

## Key Features

### üîê Enhanced DevSecOps Secrets Schema

**Structured Credential Objects**:

- Transition from flat string secrets to rich credential objects
- `type` field: api_key, password, token with smart masking behavior
- `metadata` object: created_at, expires_at, rotation_policy fields
- `rotation_policy` object: automated rotation configuration

**Lifecycle Management**:

- `created_at` and `expires_at` timestamps for credential lifecycle
- `rotation_policy` with `interval_days` and `auto_rotate` boolean
- Support for credential expiration and automated rotation workflows

**Schema Compliance**:

- Full alignment with Crucible v0.2.18 DevSecOps secrets taxonomy
- Backward compatibility with existing flat secret configurations
- Enhanced validation for structured credential objects

## Migration

### For Existing Users

No breaking changes. The enhanced DevSecOps secrets schema is backward compatible:

```go
// Existing flat secrets still work
secret := config.GetSecret("database_password")

// New structured credentials available
credential := config.GetCredential("api_key")
// credential.Type, credential.Metadata, credential.RotationPolicy
```

### For New Users

Use the enhanced structured credential objects for better security management:

```yaml
# config/crucible-go/devsecops/secrets/v1.0.0/defaults.yaml
credentials:
  api_key:
    type: api_key
    value: "${API_KEY}"
    metadata:
      created_at: "2025-11-17T00:00:00Z"
      expires_at: "2025-12-17T00:00:00Z"
    rotation_policy:
      interval_days: 30
      auto_rotate: true
```
